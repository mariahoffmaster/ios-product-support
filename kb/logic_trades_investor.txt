OVERVIEW
--------
After HEAs are originated, Unlock sells them to investors. This process is called "papering
a trade." Trades involve proposing a set of assets to an investor at an agreed price, executing
trade documents, receiving proceeds, and transferring collateral to the investor's custodian.
Key investors include Saluda Grade (SGALT B, SGLT-8), D2 (UNSO FUNDING TRUST I), LibreMax
(LEX 1, LEX 2), Canyon, and various securitization trusts (UNLOK 2022-1 through 2025-3).

KEY TERMINOLOGY
---------------
- oface — Original face value (= investment_payment at origination).
- investment_payment — Amount Unlock paid for the HEA.
- net_proceeds — Amount received from investor on settlement (after price adjustment).
- exchange_rate — The trade price / exchange rate agreed with investor.
- base_price — Starting price before adjustment. Base Exchange (NOT Exchange Rate) should
  be rounded DOWN to nearest tenth (2.05 → 2.00, 2.15 → 2.10).
- cost_limit — The investor's cost limit on the asset.
- lien_position — Position of the HEA lien (1st, 2nd, 3rd, etc.).
- buy_sell — 'B' for buy (Unlock buying back), 'S' for sell (Unlock selling to investor).
- account — The investor account the asset is in (e.g., 'UPS', 'SGALT B', 'D2').
- counterparty — The investor counterparty on the trade (e.g., 'SGALT B', 'SGLT-8', 'UF2').
- trade_date (T) — Date the trade is agreed upon.
- settle_date (S) — Date the trade financially settles (typically T+2 or T+4 business days).
- GOS — Gain on Sale. The profit from a trade = purchase_proceeds - oface.
- Available Collateral — Assets eligible to trade: must be Certified (custodian has
  verified the electronic package) AND Disbursed (CET has disbursed funds to consumer).
- Strats — Statistical stratifications report showing portfolio composition (FICO, LTV, etc.)
  generated as PDF and Excel. Used for investor reporting and trade proposals.
- Tape — Asset-level data export in Excel. Sent to investors for trade proposals and to
  custodians for certification.
- uw_issue — Underwriting issue flag on an asset. Check before including in a trade:
  =Badger.Get(B20,"uw_issue")
- is_flow_investor — Flag indicating asset is held by a flow investor (not a securitization).
- WAVG — Weighted average metrics (credit score, CLTV, ACL) calculated from Strats.
- CPR — Conditional Prepayment Rate. Measures prepayment speed.
- CDR — Conditional Default Rate. Measures default rate based on auction completions.
- 3M CPR / CDR — Trailing 3-month rates.
- LTD CPR — 24-month CPR (used as Life-to-Date proxy).
- LTD CDR — Life-to-Date CDR from trust start date.


INVESTOR SALE FLOW OF FUNDS
-----------------------------
Trade Date (T): Assets, price, trade date, and settle date are agreed upon with investor.
  In Modern Treasury: assets are marked with trade_id; expected payments are set up for
  settle date proceeds split by funding source (UPS vs. UF2).

Settle Date (S):
  1. Investor wires proceeds per split to UPS/UF2:
     - Proceeds for UPS-funded assets → wire to UPS JPM account.
     - Proceeds for UF2-funded assets → wire to UF2 TCB account.
  2. GOS (Gain on Sale) proceeds remitted from UF2 to UPS.
     Modern Treasury: expected payments reconciled; automatic GOS sweep initiated when
     UF2 funds received. Excess (GOS + haircut) swept to UPS.
  3. For assets funded via warehouse facility: 90% advance (Advance) pays down UF2 account.

Example (122 assets, trade price 104):
  Proceeds for UPS assets:  $1,804,865.92 → JPM UPS
  Proceeds for UF2 assets:  $8,863,634.00 → TCB UF2
  UF2 Advance repaid:       $7,670,452.50 (pays down warehouse line)
  Excess (GOS + Haircut):   $340,909.00 swept to UPS


PAPERING A TRADE — STEP-BY-STEP
---------------------------------
Step 1: Determine Available Collateral
  Open Trades Setup Template → Available Collateral tab.
  Click "Get Available Collateral" button. Assets must be:
    (a) Certified — electronic package verified by document custodian (WSFS).
    (b) Disbursed — CET has disbursed funds to consumer.

Step 2: Build Trade Population
  Open Forward Sale Data Schedule (copy from previous trade, update date).
  - Exclude assets in "Assets in Timeouts" section.
  - Click Money Maker to retrieve population.
  - Send proposed trade to investor (e.g., Saluda Grade email to Ben Haltmaier).
  - Copy down Base Price.
  - Adjust Base Exchange to round down to nearest tenth.
    Formula: =IF(ISODD(H20/0.05),H20-0.05,H20) then copy values back.
  - Run uw_issue check on all assets: =Badger.Get(B20,"uw_issue")
  - Fix all Pre-Execution Exceptions.

Step 3: Generate Tape
  Run New Tape Format v8.sql with appropriate parameters:
    SET @cutoff = '[settle_date]'
    declare @accounts dbo.IntList
    insert into @accounts (item) values ([account_number])
  Copy output to Excel. Remove first four columns (Account, Trade Id, Trade Date, Settle Date).
  Copy formats from previous tapes. Update Current Credit Score from Original where missing.
  Email proposed tape to investor.

Step 4: Request WSFS Certification
  Copy asset IDs into Badger search screen. Select all → right click → Admin Tasks →
  Request WSFS Certification.
  Save file as: wsfs_trade_tape_MM_DD_YYYY_Trade_[id].xlsx
  Save to: A:\00 - Operations\Document Custody\wsfs
  Send certification request email to WSFS.

Step 5: Generate Schedule and Strats
  From Schedule tab in Trades Setup Template: Run for A and then I (Spawn = Yes).
  Click "Do All" → SGALT Button (run in debug, skip strats).
  Remove notices@saludagrade.com from cc list before sending email.

Step 6: Insert Trades
  Click Insert Trade button in Trades Setup Template.
  Click Tape Icon button to generate old-format tape (required by some investors on trade date).

Step 7: DocuSign
  Location: Shared Folders\AssetManagement\[Investor]\[Investor] - Trade Package
  - Add newly generated documents (Bill of Sale, Forward Sale Data Schedule).
  - Collateral Release form completed in DocuSign package, split from other docs.
  - Send signed trade package to investor.

Step 8: On Settle Date — Run WSFS Custody Transfer
  Run WSFSCustodyToday.bat from ReportRunner.
  Send collateral release email to WSFS AFTER funds have been released.
  Email to WSFS: request for collateral transfer with relevant trade number.

Step 9: Create Position Tapes (3 total)
  Run New Tape Format v8.sql with @trade_sd=1, @include_zero_positions=1.
  Parameters for position tape:
    declare @blank_trade bit = 0
    declare @blank_price bit = 0
    declare @trade_sd bit = 1
    declare @trade_td bit = 0
  Tapes needed:
    1. Original position tape
    2. New positions with settle
    3. All-inclusive position on full portfolio

SALUDA MONTHLY TRADE STRATS
------------------------------
Frequency: Monthly. Provides Saluda with aggregate breakdown of prior month production.
Outputs: All Trades Strats (PDF), Saluda-Only Strats (PDF), Non-Saluda Strats (PDF),
WAVG Comparison Table (Excel), Tapes x3 (Excel).

Step 1: Pull asset IDs for prior month trades.
  DECLARE @asOf datetime = 'MM/01/YYYY'  -- First of prior month
  DECLARE @asOfend datetime = eomonth(@asOf)
  SELECT DISTINCT asset_id, counterparty, investment_payment
  FROM vTradesEx
  WHERE trade_date BETWEEN @asOf AND @asOfend
    AND counterparty != 'UF2' AND account = 'UPS'
  ORDER BY counterparty

  Sanity check: run volume/count query grouped by counterparty.

Step 2: Build three asset ID lists in Excel:
  - All Trades (full output)
  - Saluda-only (counterparty = 'SGALT B' or 'SGLT-8')
